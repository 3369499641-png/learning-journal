# 学习总结模仿进步 — 项目说明

## 项目定位

用户的**长线学习系统**，用于每日积累、总结、模仿、进步。核心使用场景：用户会不定期发来投资圈子的聊天记录（王世辰圈子），需要你提取知识、归档、更新网页。

## 用户偏好

- **输出简洁**：只展示问题输入和最终结果，中间过程默认折叠
- **格式偏好**：网页 > 表格/图表 > 纯文本。md 文件只是数据源，HTML 才是阅读界面
- **日期**：所有案例必须带日期，格式 `YYYY-MM-DD` 或 `M月D日`
- **自主调节**：文件夹和栏目结构可以根据实际使用情况随时调整

## 核心工作流：收到新聊天记录时

当用户发来新的交易日聊天记录时，按以下步骤处理：

### 1. 保存原始记录
在 `05_参考资料/文章/投资圈聊天记录/` 下创建 `YYYY-MM-DD.md`，存原文。

### 2. 提取新案例
在聊天中识别新股票/新操作逻辑 → 添加到 `03_个股案例库.md` 和 `index.html` 的案例卡片区。
- 格式：`案例 N：股票名 — 核心逻辑`，必须带日期
- 每案例含：时间、逻辑、操作、经验四段

### 3. 提取新知识点
- 新看盘技巧 → 追加到 `01_看盘框架.md` 和 HTML 对应 section
- 新战法 → 追加到 `02_交易战法.md` 和 HTML 对应 section
- 新金句/纪律 → 追加到 `04_心法与纪律.md` 和 HTML 对应 section

### 4. 创建每日记录
在 `01_每日记录/` 创建 `YYYY-MM-DD_主题.md`，用 `_模板.md` 的格式。

### 5. 同步 HTML
**这是最关键的一步**。index.html 是用户实际阅读的界面，每次更新 md 后必须同步更新 HTML 中对应的 section。

### 6. 部署到 GitHub Pages

**如果你拿到的是 zip 包（不含 .git），必须先初始化 git：**

```bash
# 第一步：进入项目目录
cd "C:/Users/YZ/Desktop/学习总结模仿进步"

# 第二步：初始化 git 并关联远程仓库
git init
git remote add origin https://github.com/3369499641-png/learning-journal.git
git fetch origin main
git checkout -b main origin/main

# 第三步：确保 gh CLI 可用（需要先安装：winget install GitHub.cli）
export PATH="$PATH:/c/Program Files/GitHub CLI"
gh auth status
# 如果未登录，运行 gh auth login --web

# 如果 gh auth setup-git 还没跑过，先配置 git 凭证：
gh auth setup-git

# 第四步：正常提交流程
git add -A
git commit -m "feat: YYYY-MM-DD 交易日更新"
git push origin main
```

**如果 git push 超时（Windows 上常见），用 API 逐文件上传：**
```bash
# 读文件 → base64 → API PUT
CONTENT=$(base64 -w0 要上传的文件)
gh api repos/3369499641-png/learning-journal/contents/路径/文件名 \
  -X PUT -f "message=commit消息" -f "content=$CONTENT" -f "branch=main"
```

推完后触发 Pages 重建：
```bash
gh api repos/3369499641-png/learning-journal/pages/builds -X POST
```

## HTML 文件结构说明

`02_专题学习/投资/index.html` 是单页应用，结构：
- **侧边栏**：`#sidebar` → 导航链接（`#section-1` ~ `#section-4`）
- **搜索**：`#search` → 实时过滤
- **Section 1**：看盘框架 → id="section-1"
- **Section 2**：交易战法 → id="section-2"
- **Section 3**：个股案例库 → id="section-3"（卡片用 `.card-grid > .card`）
- **Section 4**：心法与纪律 → id="section-4"

**添加新案例卡片模板：**
```html
<div class="card">
  <h4>案例 N：股票名</h4>
  <p><span class="tag">日期</span> <span class="tag">题材标签</span></p>
  <p><strong>逻辑</strong>：...</p>
  <p><strong>操作</strong>：...</p>
  <p style="margin-top:8px"><strong>经验</strong>：...</p>
</div>
```

## 用户的目标

通过系统化整理投资圈子的知识，逐步建立自己的交易体系和看盘能力。最终目的是：看懂盘面 → 用对战法 → 管住手 → 控制回撤 → 稳定盈利。

## 对其他 AI 的建议

1. **优先更新 HTML**：用户用浏览器阅读，HTML 是第一界面
2. **保持风格一致**：案例格式、术语、排版风格
3. **日期绝不遗漏**：每个案例、每条记录都要标注日期
4. **可自主扩展**：专题不限于投资，用户后续可能新增编程、写作等方向
5. **简洁输出**：更新完成后只告知改动摘要，不展开全部内容
